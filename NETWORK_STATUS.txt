  THE TECHNICAL CHALLENGE


The libp2p Swarm is !Send (contains trait objects that aren't
thread-safe). This means we can't simply spawn it in a tokio
task. We have a few options:

OPTION 1: Run network loops in LocalSet
   Requires all async tasks in same thread
   Works for tests, harder for production

OPTION 2: Message-passing architecture
   Send commands to network thread via channel
   Network thread owns the Swarm
   More complex but production-ready

OPTION 3: Accept manual polling in tests
   Tests drive network manually
   Real app would run dedicated network thread
   Simpler, proves CRDT logic works
